Trade Management Online

System Architecture

Front Ends

  • Two projects(TMO and TMW) each serving its own purpose
  • These projects are built using the Angular (Version 4) MVVC Framework
  • Solar based on Bootstrap library
  • Uses NPM to manage its dependencies
  • Karma and Jasmine ( Unit testing frameworks) used to execute unit tests
  • Finally, Webpack is used to bundle the entire application into a single executable package

Backend

  • Spring Boot Application using Maven as its dependency management system
  • Used to provide Restful API to service the front ends
  • Data is persisted and retrieved from a SQL Server Database
  • Developed using a Domain Driven Design architecture paradigm while making use of CQRS and event sourcing. Axon (version 3) framework is used to make this possible
  • A dockerised image of the app is created during the final build and hosted in Openshift

WebSlides is really easy

Each parent <section> in the #webslides element is an individual slide.

Code is clean, scalable, and well documented. It uses intuitive markup with popular naming conventions. There's no need to overuse classes or nesting. Based on SimpleSlides, by Jenn Schiffer :)

<article id="webslides">
  <!-- Slide 1 -->
  <section>
    <h1>Design for trust</h1>
  </section>
  <!-- Slide 2 -->
  <section class="bg-primary">
    <div class="wrap">
      <h2>.wrap = container (width: 90%)</h2>
    </div>
  </section>
</article>
